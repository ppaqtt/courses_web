    1. TypeScript简介

    📖 知识点讲解

    • 什么是TypeScript：TypeScript是微软开发的开源编程语言，是JavaScript的超集，添加了可选的静态类型和基于类的面向对象编程
    • 核心优势：编译时类型检查、更好的IDE支持（智能提示、重构）、增强代码可读性与可维护性
    • 与JavaScript关系：TS完全兼容JS，任何合法的JS代码都是合法的TS代码，TS最终编译为JS运行
    • 适用场景：大型项目开发、团队协作、需要长期维护的企业级应用、前端框架（Angular/Vue/React）
    typescript 
    // TypeScript为JavaScript添加了类型系统
let username: string = "TypeScript";
let version: number = 5.0;
let isOpenSource: boolean = true;

// 类型推断：TypeScript可以自动推断类型
let message = "Hello, TypeScript!"; // 推断为string类型

console.log(`${username} v${version}`);
console.log(`开源: ${isOpenSource}`);
console.log(message);

    ❌ 常见错误

    类型不匹配：let age: string = 25; 将number赋值给string类型会报错，应改为 let age: number = 25;

    认为TS是全新语言：TypeScript是JavaScript的超集，不是替代品，所有JS代码都可以直接在TS中使用

    2. 开发环境配置

    📖 知识点讲解

    • 安装Node.js：TypeScript运行在Node.js环境中，需先安装Node.js（建议LTS版本）
    • 安装TypeScript：使用npm全局安装 npm install -g typescript，安装后可通过 tsc -v 验证
    • 编辑器推荐：VS Code是开发TS的首选编辑器，内置TypeScript支持，提供智能提示和错误检查
    • 在线工具：TypeScript Playground（www.typescriptlang.org/play）可在线编写和运行TS代码
    bash 
    # 安装TypeScript编译器
npm install -g typescript

# 验证安装
tsc -v

# 创建项目目录
mkdir my-ts-project && cd my-ts-project

# 初始化项目
npm init -y

# 安装本地TypeScript
npm install --save-dev typescript

# 创建tsconfig.json
tsc --init

    ❌ 常见错误

    tsc不是内部命令：未正确安装或PATH未配置，重新运行 npm install -g typescript 并确认Node.js已安装

    全局与本地版本冲突：建议项目使用本地安装的TS，通过 npx tsc 运行

    3. 第一个TS程序

    📖 知识点讲解

    • 创建TS文件：TypeScript文件以 .ts 为扩展名，可以包含类型注解和ES6+语法
    • 编译TS：使用 tsc 文件名.ts 编译为JS，生成同名 .js 文件
    • 类型注解：使用 : 类型 为变量添加类型约束，如 let name: string
    • 编译运行：也可使用 ts-node 直接运行TS文件，无需手动编译
    typescript 
    // hello.ts - 第一个TypeScript程序
function greet(name: string): string {
    return `你好, ${name}! 欢迎学习TypeScript!`;
}

const user: string = "开发者";
console.log(greet(user));

// 类型注解帮助我们在编译时发现错误
// greet(42); // 错误：Argument of type 'number' is not assignable to parameter of type 'string'

    ❌ 常见错误

    直接运行TS文件：node hello.ts 会报错，应先编译 tsc hello.ts && node hello.js 或使用 ts-node hello.ts

    类型不匹配传参：向需要string的函数传入number类型，编译器会直接报错

    4. 编译与tsconfig

    📖 知识点讲解

    • tsconfig.json：TypeScript项目的配置文件，使用 tsc --init 生成，定义编译选项和项目结构
    • 常用编译选项：target（编译目标ES版本）、strict（严格模式）、module（模块系统）、outDir（输出目录）
    • 严格模式：启用 strict: true 会开启所有严格类型检查选项，推荐在新项目中使用
    • 监听模式：使用 tsc -w 或 tsc --watch 监听文件变化自动重新编译
    json 
    {
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "strict": true,
    "outDir": "./dist",
    "rootDir": "./src",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "sourceMap": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}

    ❌ 常见错误

    缺少tsconfig.json：直接运行 tsc 时如果缺少配置文件，会使用默认设置，可能导致输出目录混乱

    未开启strict：不开启严格模式会错过许多类型检查，建议始终设置 "strict": true

    📝 练习题

    1. 哪个编译选项用于指定编译输出的ES版本？
    B. target（如 "target": "ES2020"）

    5. 基础类型(number/string/boolean)

    📖 知识点讲解

    • number类型：所有数字都是number类型，包括整数和浮点数，TS没有int/float之分，也支持二进制(0b)、八进制(0o)、十六进制(0x)
    • string类型：使用单引号、双引号或反引号（模板字符串），模板字符串支持多行和嵌入表达式
    • boolean类型：只有 true 和 false 两个值，注意真值/假值与布尔类型的区别
    • 类型推断：当变量声明时赋值，TS会自动推断类型，无需显式注解
    typescript 
    // number类型
let age: number = 25;
let price: number = 99.99;
let hex: number = 0xff;      // 十六进制
let binary: number = 0b1010; // 二进制

// string类型
let name: string = "TypeScript";
let greeting: string = `Hello, ${name}!`; // 模板字符串

// boolean类型
let isActive: boolean = true;
let hasPermission: boolean = false;

// 类型推断 - 以下写法TS会自动推断类型
let count = 10;        // 推断为 number
let title = "Hello";   // 推断为 string
let enabled = true;    // 推断为 boolean

console.log(`${name}, 年龄: ${age}, 活跃: ${isActive}`);

    ❌ 常见错误

    类型赋值错误：let age: number = "25"; 字符串不能赋值给number类型，应使用 Number("25") 或 parseInt("25")

    混淆null和boolean：let flag: boolean = 0; 0不是boolean，需显式 Boolean(0) 或 !!0

    📝 练习题

    1. TypeScript中整数和浮点数是否属于不同类型？
    B. 否，都是number类型（TypeScript与JavaScript一致，不区分整数和浮点数）

    6. 数组与元组

    📖 知识点讲解

    • 数组类型：两种写法 number[] 或 Array<number>，推荐使用前者
    • 元组类型：固定长度和类型的数组，每个位置的类型可以不同，如 [string, number]
    • 只读数组：使用 readonly number[] 或 ReadonlyArray<number> 创建不可变数组
    • 解构赋值：数组和元组都支持解构，元组解构时每个变量都有对应的类型
    typescript 
    // 数组
let numbers: number[] = [1, 2, 3, 4, 5];
let names: Array<string> = ["Alice", "Bob", "Charlie"];

// 元组 - 固定长度和类型
let person: [string, number] = ["张三", 25];
let rgb: [number, number, number] = [255, 128, 0];

// 元组解构
const [name, age] = person;
console.log(`姓名: ${name}, 年龄: ${age}`);

// 只读数组
const readOnlyArr: readonly number[] = [1, 2, 3];
// readOnlyArr.push(4); // 错误：不允许修改

// 可变元组
type StringNumberBooleans = [string, number, ...boolean[]];
let mixed: StringNumberBooleans = ["hello", 42, true, false, true];

    ❌ 常见错误

    元组越界访问：let t: [string, number] = ["hi", 1]; t[2] = true; 在严格模式下会报错，元组长度已固定

    混合类型数组：let arr = [1, "two"]; 会被推断为 (string | number)[] 而非元组

    📝 练习题

    1. 定义一个表示坐标(x, y)的元组类型？
    type Coordinate = [number, number]; let point: Coordinate = [10, 20];

    7. 枚举类型

    📖 知识点讲解

    • 数字枚举：默认从0开始自增，可以手动指定初始值，支持反向映射（从值到名）
    • 字符串枚举：每个成员必须显式赋值字符串，不支持反向映射
    • 常量枚举：使用 const enum 定义，编译时内联，不会生成额外JS代码
    • 异构枚举：混合数字和字符串成员（不推荐），容易产生混淆
    typescript 
    // 数字枚举
enum Direction {
    Up = 1,
    Down,   // 2
    Left,   // 3
    Right   // 4
}
let dir: Direction = Direction.Up;

// 字符串枚举
enum UserRole {
    Admin = "ADMIN",
    Editor = "EDITOR",
    Viewer = "VIEWER"
}
let role: UserRole = UserRole.Admin;

// 常量枚举 - 编译时内联
const enum Color {
    Red,
    Green,
    Blue
}
let c: Color = Color.Green; // 编译后直接替换为 1

console.log(`方向: ${dir}, 角色: ${role}, 颜色值: ${c}`);

    ❌ 常见错误

    字符串枚举未赋值：enum E { A = "hello", B } 字符串枚举成员B必须显式赋值

    常量枚举的坑：const enum 在 isolatedModules 模式下不能使用，且无法进行反向映射

    📝 练习题

    1. 数字枚举默认从几开始编号？
    0（数字枚举默认从0开始自增）

    📒 学习笔记

    8. any与unknown

    📖 知识点讲解

    • any类型：放弃类型检查，允许赋值为任何类型，也能赋值给其他类型，应尽量避免使用
    • unknown类型：类型安全的any，可以接收任何值，但使用前必须进行类型检查/类型收窄
    • 类型收窄：使用 typeof、instanceof 或类型断言来收窄unknown类型
    • 选择建议：优先使用unknown代替any，保留类型安全的同时提供灵活性
    typescript 
    // any - 放弃类型检查（不推荐）
let notSure: any = 4;
notSure = "maybe a string";
notSure = false; // 都不会报错

// unknown - 类型安全的顶类型（推荐）
let value: unknown = 42;
value = "hello";
value = true;

// 使用unknown前必须类型收窄
function processValue(val: unknown) {
    if (typeof val === "string") {
        console.log(val.toUpperCase()); // OK: 收窄为string
    } else if (typeof val === "number") {
        console.log(val.toFixed(2));    // OK: 收窄为number
    } else {
        console.log("未知类型");
    }
}

processValue("typescript");
processValue(3.14159);

    ❌ 常见错误

    直接使用unknown：let v: unknown = "hi"; v.toUpperCase(); 编译报错，必须先收窄类型

    滥用any：将所有不确定的类型标记为any，等于放弃了TS的类型保护，应使用unknown替代

    📝 练习题

    1. any和unknown的关键区别是什么？
    any可以任意赋值和使用（无类型检查），unknown可以接收任意值但使用前必须类型收窄

    📒 学习笔记

    9. void与never

    📖 知识点讲解

    • void类型：表示函数没有返回值，只能赋值为 undefined 或 null（strictNullChecks关闭时）
    • never类型：表示永远不会出现的值，如抛出异常的函数或无限循环的函数的返回类型
    • never的应用：类型收窄中的穷尽检查（switch中的default分支）、不可达代码标注
    • 区别：void是"没有返回值"，never是"永远不会有返回值"，never是所有类型的子类型
    typescript 
    // void - 函数没有返回值
function logMessage(msg: string): void {
    console.log(msg);
}

// never - 永远不会有返回值
function throwError(msg: string): never {
    throw new Error(msg);
}

function infiniteLoop(): never {
    while (true) {}
}

// never在类型收窄中的应用
type Shape = "circle" | "square";
function getArea(shape: Shape) {
    switch (shape) {
        case "circle": return Math.PI;
        case "square": return 1;
        default:
            const _exhaustive: never = shape; // 如果有遗漏会报错
            return _exhaustive;
    }
}

logMessage("void示例");
console.log(getArea("circle"));

    ❌ 常见错误

    混淆void和undefined：函数返回 undefined 和 void 不同，void表示"不需要关心返回值"

    给never赋值：let x: never = 1; 报错，never不能被赋任何值（除了never本身）

    📝 练习题

    1. 一个总是抛出异常的函数应该返回什么类型？
    never（因为函数永远不会正常返回）

    10. 类型断言

    📖 知识点讲解

    • 尖括号语法：<string>value，在JSX中不可用，与React语法冲突
    • as语法：value as string，推荐写法，JSX中也可使用
    • 双重断言：value as unknown as string，强制类型转换（极度不推荐）
    • 非空断言：value! 告诉编译器该值不为null/undefined，运行时仍可能出错
    typescript 
    // as语法（推荐）
let someValue: unknown = "Hello TypeScript";
let strLength: number = (someValue as string).length;

// 尖括号语法（JSX中不可用）
let value: unknown = "hello";
let len: number = (<string>value).length;

// 非空断言 !
function getElement(id: string): HTMLElement | null {
    return document.getElementById(id);
}
// 告诉TS此元素一定存在
const el = getElement("app")!;

// const断言
const obj = { x: 10, y: 20 } as const; // 变为readonly
// obj.x = 5; // 错误：readonly

console.log(`字符串长度: ${strLength}`);

    ❌ 常见错误

    滥用类型断言：let n = "hello" as number; 不兼容的类型不能直接断言，需要先转为unknown

    非空断言隐患：el!.style.color = "red"; 如果el实际为null，运行时会崩溃

    📝 练习题

    1. 在JSX/TSX文件中，应该使用哪种类型断言语法？
    as语法（如 value as string），因为尖括号语法与JSX标签冲突

    11. 接口定义

    📖 知识点讲解

    • 接口定义：使用 interface 关键字定义对象的形状（shape），描述对象应该有哪些属性和方法
    • 可选属性：使用 ? 标记，如 age?: number，该属性可以不存在
    • 只读属性：使用 readonly 修饰，只能在创建时赋值，之后不可修改
    • 函数类型：接口可以描述函数类型，定义参数和返回值的类型
    typescript 
    // 基本接口定义
interface User {
    name: string;
    age: number;
    email: string;
}

const user: User = {
    name: "张三",
    age: 25,
    email: "zhangsan@example.com"
};

// 可选属性与只读属性
interface Config {
    readonly id: number;
    name: string;
    description?: string; // 可选
}

const config: Config = { id: 1, name: "app" };
// config.id = 2; // 错误：readonly属性不可修改

console.log(`用户: ${user.name}, 配置: ${config.name}`);

    ❌ 常见错误

    缺少必需属性：const u: User = { name: "a" }; 缺少age和email属性会报错

    多余属性：直接赋值对象字面量时不能有多余属性，但赋值给变量后再传入则不会报错（结构化类型）

    📝 练习题

    1. 定义一个Book接口，包含title(string)、author(string)、pages?(可选number)属性？
    interface Book { title: string; author: string; pages?: number; }

    📒 学习笔记

    12. 可选与只读属性

    📖 知识点讲解

    • 可选属性(?)：属性可以不存在，访问时类型为 T | undefined，需注意可能为undefined的情况
    • 只读属性(readonly)：只能在对象初始化时赋值，之后不可修改，保护关键数据不被意外篡改
    • Readonly工具类型：Readonly<T> 将所有属性变为只读，比逐个添加readonly更方便
    • const与readonly区别：const用于变量声明，readonly用于属性声明
    typescript 
    interface Point {
    readonly x: number;
    readonly y: number;
    label?: string; // 可选属性
}

const p: Point = { x: 10, y: 20 };
// p.x = 5; // 错误：readonly

// 可选属性需要判断
if (p.label) {
    console.log(p.label.toUpperCase());
}

// Readonly工具类型
interface MutableUser {
    name: string;
    age: number;
}
type ReadOnlyUser = Readonly<MutableUser>;
const ro: ReadOnlyUser = { name: "A", age: 20 };
// ro.name = "B"; // 错误：readonly

    ❌ 常见错误

    直接访问可选属性：p.label.toUpperCase() 当label为undefined时会崩溃，应使用可选链 p.label?.toUpperCase()

    readonly不等于深度不可变：readonly 只是一层，对象内部属性仍可修改

    📝 练习题

    1. 如何安全地访问一个可选属性？
    使用可选链操作符 obj.prop?.method() 或先判断 if (obj.prop) {...}

    📒 学习笔记

    13. 接口继承

    📖 知识点讲解

    • 单继承：使用 extends 继承一个接口，子接口拥有父接口的所有属性
    • 多继承：接口可以同时继承多个接口，用逗号分隔
    • 重写属性：子接口可以重新定义父接口的属性类型，但必须兼容（协变）
    • 与类型别名的区别：interface可继承和被继承，type只能通过交叉类型实现类似效果
    typescript 
    // 单继承
interface Animal {
    name: string;
    age: number;
}

interface Dog extends Animal {
    breed: string;
    bark(): void;
}

const myDog: Dog = {
    name: "旺财",
    age: 3,
    breed: "金毛",
    bark() { console.log("汪汪！"); }
};

// 多继承
interface Serializable {
    serialize(): string;
}

interface Loggable {
    log(): void;
}

interface Entity extends Serializable, Loggable {
    id: number;
}

myDog.bark();

    ❌ 常见错误

    继承后缺少属性：实现子接口时必须包含父接口的所有必需属性

    不兼容重写：将父接口的 name: string 重写为 name: number 会报错，类型不兼容

    📝 练习题

    1. 接口可以同时继承多个接口吗？
    A. 可以（interface A extends B, C, D {...}）

    📒 学习笔记

    14. 函数类型接口

    📖 知识点讲解

    • 函数类型定义：使用接口定义函数的参数和返回值类型
    • 函数重载：同一函数名多个签名，TypeScript会从上到下匹配
    • 可选参数与默认参数：可选参数用?，默认参数直接赋默认值
    • rest参数：使用 ...args 收集剩余参数，类型为数组
    typescript 
    // 函数类型接口
interface MathFunc {
    (x: number, y: number): number;
}

const add: MathFunc = (a, b) => a + b;
const multiply: MathFunc = (a, b) => a * b;

// 函数重载
function padLeft(value: string, padding: number): string;
function padLeft(value: string, padding: string): string;
function padLeft(value: string, padding: number | string): string {
    if (typeof padding === "number") {
        return " ".repeat(padding) + value;
    }
    return padding + value;
}

// 可选参数与默认值
function greet(name: string, greeting?: string): string {
    return `${greeting || "你好"}, ${name}!`;
}

// rest参数
function sum(...nums: number[]): number {
    return nums.reduce((a, b) => a + b, 0);
}

console.log(add(3, 5));       // 8
console.log(greet("TypeScript")); // 你好, TypeScript!
console.log(sum(1, 2, 3, 4)); // 10

    ❌ 常见错误

    可选参数位置：可选参数不能放在必需参数前面，function f(a?: string, b: number) 报错

    重载签名不匹配：实现签名必须兼容所有重载签名

    15. 类定义

    📖 知识点讲解

    • 类的基本结构：TypeScript类支持属性声明、构造函数、方法，属性需先声明后使用
    • 构造函数：使用 constructor 初始化实例，支持参数属性简写

    16. 访问修饰符

    📖 知识点讲解

    • public：默认修饰符，任何地方都可以访问
    • private：只在类内部可访问，子类和外部都不能访问
    • protected：类内部和子类可访问，外部不可访问
    • readonly：修饰属性为只读，只能在声明时或构造函数中赋值
    typescript 
    class BankAccount {
    public owner: string;        // 公开
    private balance: number;     // 私有
    protected accountId: string; // 受保护
    readonly createdAt: Date;    // 只读

    constructor(owner: string, balance: number) {
        this.owner = owner;
        this.balance = balance;
        this.accountId = `ACC-${Date.now()}`;
        this.createdAt = new Date();
    }

    public getBalance(): number {
        return this.balance; // 内部可以访问private
    }

    public deposit(amount: number): void {
        if (amount > 0) this.balance += amount;
    }
}

class SavingsAccount extends BankAccount {
    constructor(owner: string, balance: number) {
        super(owner, balance);
        // this.balance; // 错误：private不可在子类访问
        console.log(this.accountId); // OK：protected可以在子类访问
    }
}

const acc = new BankAccount("张三", 1000);
console.log(acc.owner);        // OK
// console.log(acc.balance);   // 错误：private不可外部访问

    ❌ 常见错误

    外部访问private：acc.balance 报错，private成员只能通过公开方法间接访问

    混淆private和protected：private子类也不能访问，protected子类可以访问

    17. 继承与重写

    📖 知识点讲解

    • 类继承：使用 extends 继承父类，子类获得父类所有public和protected成员
    • 方法重写：子类可以重写父类方法，使用 super 调用父类版本
    • 构造函数重写：子类构造函数必须调用 super()，且在使用this之前
    • 类型兼容性：子类实例可以赋值给父类类型的变量（里氏替换原则）
    typescript 
    class Animal {
    constructor(public name: string) {}

    move(distance: number = 0): void {
        console.log(`${this.name} 移动了 ${distance} 米`);
    }

class Snake extends Animal {
    constructor(name: string) {
        super(name); // 必须先调用super
    }

    move(distance: number = 5): void {
        console.log("滑行中...");
        super.move(distance); // 调用父类方法
    }
}

class Horse extends Animal {
    constructor(name: string) {
        super(name);
    }

    move(distance: number = 50): void {
        console.log("奔跑中...");
        super.move(distance);
    }
}

sam.move();
tom.move(100);

    忘记super()：子类构造函数中使用this前必须先调用 super()

    重写签名不兼容：子类方法参数类型必须兼容父类（协变/逆变规则）

    18. 抽象类

    📖 知识点讲解

    • 抽象类定义：使用 abstract 修饰的类，不能直接实例化，只能被继承
    • 抽象方法：只有签名没有实现，子类必须实现所有抽象方法
    • 具体方法：抽象类也可以包含具体实现，子类可以直接使用或重写
    • 与接口的区别：抽象类可以有实现代码，接口只有声明；类只能继承一个抽象类，但可实现多个接口
    typescript 
    abstract class Shape {
    constructor(public color: string) {}

    // 抽象方法 - 子类必须实现
    abstract getArea(): number;
    abstract getPerimeter(): number;

    // 具体方法 - 子类可直接使用
    describe(): string {
        return `这是一个${this.color}的形状，面积=${this.getArea().toFixed(2)}`;
    }
}

class Circle extends Shape {
    constructor(color: string, public radius: number) {
        super(color);
    }

    getArea(): number {
        return Math.PI * this.radius ** 2;
    }

    getPerimeter(): number {
        return 2 * Math.PI * this.radius;
    }
}

class Rectangle extends Shape {
    constructor(color: string, public width: number, public height: number) {
        super(color);
    }

    getArea(): number {
        return this.width * this.height;
    }

    getPerimeter(): number {
        return 2 * (this.width + this.height);
    }
}

const circle = new Circle("红色", 5);
console.log(circle.describe());

    ❌ 常见错误

    直接实例化抽象类：new Shape("红") 报错，抽象类不能直接实例化

    未实现抽象方法：子类如果不实现所有抽象方法，自身也必须声明为abstract

    19. 泛型函数

    📖 知识点讲解

    • 泛型函数：使用类型参数 <T> 定义可复用的类型安全函数，T在调用时确定
    • 多个类型参数：可以定义多个类型参数 <T, U>，适用于多参数函数
    • 类型推断：调用泛型函数时通常无需显式指定类型，TS会根据参数自动推断
    • 泛型默认值：<T = string> 为类型参数设置默认类型
    typescript 
    // 基本泛型函数
function identity<T>(arg: T): T {
    return arg;
}

    return { ...a, ...b };

const merged = merge({ name: "TS" }, { version: 5 });

    return Array(length).fill(value ?? 0);
}

const arr1 = createArray(3);       // number[]
const arr2 = createArray(3, "hi"); // string[]

    20. 泛型接口与类

    📖 知识点讲解

    • 泛型接口：接口使用类型参数，如 interface Repository<T>，可用于定义通用数据结构
    • 泛型类：类使用类型参数，如 class DataStore<T>，实例化时确定具体类型
    • 泛型约束接口：接口本身可以约束泛型参数，如 T extends Serializable
    • 工厂模式：泛型类常与工厂模式结合，创建类型安全的对象
    typescript 
    // 泛型接口
interface Repository<T> {
    getAll(): T[];
    getById(id: number): T | undefined;
    add(item: T): void;
}

class DataStore<T> {
    private items: T[] = [];

    add(item: T): void {
        this.items.push(item);
    }

    getAll(): T[] {
        return [...this.items];
    }

    find(predicate: (item: T) => boolean): T | undefined {
        return this.items.find(predicate);
    }
}

    21. 泛型约束

    📖 知识点讲解

    • extends约束：使用 T extends Interface 限制泛型参数必须满足特定条件
    • keyof约束：T extends keyof U 确保T是U的键名类型
    • 构造函数约束：new (...args: any[]) => T 约束泛型为可实例化的类
    • 多重约束：使用交叉类型实现 T extends A & B
    typescript 
    // 基本约束
interface HasLength {
    length: number;
}

function logLength<T extends HasLength>(arg: T): void {

    23. 交叉类型

    📖 知识点讲解

    • 交叉类型定义：使用 & 将多个类型合并为一个类型，新类型具有所有成员
    • 与联合类型区别：联合类型是"或"关系（取并集的属性），交叉类型是"且"关系（取交集的属性）
    • 冲突处理：相同属性名但类型不同时，结果类型为 never
    • 常见用途：Mixin模式、组合多个接口、添加额外属性
    typescript 
    // 基本交叉类型
interface BusinessPartner {
    name: string;
    credit: number;
}

interface Identity {
    id: number;
    email: string;
}

type Employee = BusinessPartner & Identity;

const emp: Employee = {
    name: "张三",
    credit: 1000,
    id: 1,
    email: "zhangsan@company.com"
};

// Mixin模式
type WithTimestamp<T> = T & { createdAt: Date };
type WithVersion<T> = T & { version: number };

interface Doc {
    title: string;
    content: string;
}

type VersionedDoc = WithTimestamp<Doc> & WithVersion<Doc>;

const doc: VersionedDoc = {
    title: "报告",
    content: "内容...",
    createdAt: new Date(),
    version: 1
};

console.log(`${emp.name} - ${doc.title}`);

    ❌ 常见错误

    类型冲突：type A = { x: string } & { x: number } 中x的类型变为never，实际无法使用

    缺少必需属性：交叉类型要求满足所有类型的全部属性

    📝 练习题

    1. 联合类型(|)和交叉类型(&)的核心区别是什么？

    24. 类型别名

    📖 知识点讲解

    • type定义：使用 type 关键字为任何类型创建别名，不创建新类型

    26. 可辨识联合

    📖 知识点讲解

    • 可辨识联合：联合类型中的每个变体都有一个共有属性（标签/判别式）用于区分类型
    • 标签属性：通常使用字面量类型作为标签，如 kind: "circle"
    • 穷尽检查：在switch中利用标签属性收窄类型，确保处理所有变体
    • 应用场景：状态机、Redux action类型、AST节点、表单验证
    typescript 
    interface Circle {
    kind: "circle";
    radius: number;
}

interface Rectangle {
    kind: "rectangle";
    width: number;
    height: number;
}

interface Triangle {
    kind: "triangle";
    base: number;
    height: number;
}

type Shape = Circle | Rectangle | Triangle;

function getArea(shape: Shape): number {
    switch (shape.kind) {
        case "circle":
            return Math.PI * shape.radius ** 2;
        case "rectangle":
            return shape.width * shape.height;
        case "triangle":
            return 0.5 * shape.base * shape.height;
    }
}

const shapes: Shape[] = [
    { kind: "circle", radius: 5 },
    { kind: "rectangle", width: 4, height: 6 },
    { kind: "triangle", base: 3, height: 8 }
];

shapes.forEach(s => console.log(getArea(s).toFixed(2)));

    ❌ 常见错误

    缺少标签属性：如果联合类型没有共有的判别式属性，就无法安全收窄

    遗漏分支：switch没有覆盖所有变体时，函数可能返回undefined（非穷尽检查）

    📝 练习题

    1. 可辨识联合的核心要素是什么？
    每个变体都有一个共有的标签属性（判别式），使用字面量类型来区分不同变体

    27. 类型守卫

    📖 知识点讲解

    • typeof守卫：typeof x === "string" 在运行时检查并收窄类型
    • instanceof守卫：x instanceof Date 检查实例类型，适用于类
    • in守卫："prop" in obj 检查属性是否存在，适用于区分接口
    • 自定义类型谓词：function isString(x: any): x is string 定义自己的类型检查函数
    typescript 
    // typeof守卫
function double(value: string | number): string | number {
    if (typeof value === "string") {
        return value.repeat(2); // 收窄为string
    }
    return value * 2; // 收窄为number
}

// instanceof守卫
function formatValue(value: Date | string) {
    if (value instanceof Date) {
        return value.toISOString();
    }
    return value.toUpperCase();
}

// in守卫
interface Fish { swim(): void }
interface Bird { fly(): void }
function move(animal: Fish | Bird) {
    if ("swim" in animal) {
        animal.swim();
    } else {
        animal.fly();
    }
}

// 自定义类型谓词
function isNonNull<T>(value: T): value is NonNullable<T> {
    return value !== null && value !== undefined;
}

const val: string | null = "hello";
if (isNonNull(val)) {
    console.log(val.toUpperCase()); // val: string
}

    typeof局限性：typeof null === "object" 和 typeof [] === "object"，不适合检查这些类型

    类型谓词返回值：类型谓词函数必须返回boolean，类型收窄依赖返回值

    28. 模块导入导出

    📖 知识点讲解

    • 命名导出：使用 export 导出变量、函数、类、接口等，可多个
    • 默认导出：使用 export default 每个模块只能有一个
    • 重导出：使用 export { X } from "./module" 从另一个模块重新导出

    typescript

export function add(a: number, b: number): number {
    return a + b;
}

export interface CalcResult {
    value: number;
    timestamp: Date;
}

export default class Logger {

}

import { add, PI } from "./math";

// 默认导入

// 类型导入（编译后移除）

    29. 命名空间

    📖 知识点讲解

    • 命名空间定义：使用 namespace 关键字组织代码，避免全局命名冲突
    • 嵌套命名空间：命名空间可以嵌套，使用 . 访问嵌套成员
    • 命名空间合并：同名的命名空间声明会自动合并（与interface类似）
    • 与模块的选择：现代TS开发推荐使用ES模块，命名空间主要用于旧代码和声明文件
    typescript 
    namespace Utils {
    export function log(msg: string): void {
        console.log(`[LOG] ${msg}`);
    }

    export function format(template: string, ...args: any[]): string {
        return args.reduce((str, arg, i) => {
            return str.replace(`{${i}}`, String(arg));
        }, template);
    }
}

// 嵌套命名空间
namespace App {
    export namespace Models {
        export interface User {
            id: number;
            name: string;
        }
    }

    export namespace Services {
        export function getUser(id: number): Models.User {
            return { id, name: `用户${id}` };
        }
    }
}

// 使用
Utils.log("命名空间示例");
const msg = Utils.format("你好, {0}! 今年{1}岁", "TypeScript", 10);
console.log(msg);

const user = App.Services.getUser(1);
console.log(user.name);

    ❌ 常见错误

    忘记export：命名空间内的成员默认不导出，外部无法访问，需加 export

    混用模块和命名空间：不应在模块内使用命名空间，二者是不同的组织方式

    📝 练习题

    1. 现代TypeScript项目中，推荐使用模块还是命名空间？
    推荐使用ES模块（import/export），命名空间主要用于旧代码兼容和声明文件

    30. 装饰器基础

    📖 知识点讲解

    • 装饰器定义：一种特殊声明，以@expression形式附加到类声明、方法、属性上，用于修改或增强行为
    • 类装饰器：应用于类构造函数，可以修改类定义或替换类
    • 方法装饰器：应用于方法，可以修改方法行为，如日志、权限检查等
    • 启用装饰器：需在tsconfig.json中设置 "experimentalDecorators": true
    typescript
    // 类装饰器
function Logger(target: Function) {
    console.log(`创建类: ${target.name}`);
}

@Logger
class MyClass {
    constructor() { console.log("实例化"); }
}

// 方法装饰器
function Log(target: any, key: string, descriptor: PropertyDescriptor) {
    const original = descriptor.value;
    descriptor.value = function(...args: any[]) {
        console.log(`调用 ${key}，参数: ${args}`);
        return original.apply(this, args);
    };
}

class Calculator {
    @Log
    add(a: number, b: number): number {
        return a + b;
    }
}

const c = new Calculator();
c.add(2, 3);

    ❌ 常见错误

    未启用装饰器：忘记在tsconfig.json中设置 "experimentalDecorators": true 会导致编译错误

    装饰器执行时机：装饰器在类定义时执行，而非实例化时

    📝 练习题

    1. 使用TypeScript装饰器需要启用哪个编译选项？
    experimentalDecorators（需在tsconfig.json中设为true）

    32. 实用类型工具(Partial/Required/Pick)

    📖 知识点讲解

    • Partial<T>：将T的所有属性变为可选，常用于更新函数的参数类型
    • Required<T>：将T的所有属性变为必需，与Partial相反
    • Pick<T, K>：从T中选取部分属性组成新类型
    • Omit<T, K>：从T中排除部分属性，与Pick互补；Record<K, V>创建键为K值为V的对象类型
    typescript
    interface User {
    id: number;
    name: string;
    email: string;
    age: number;
}

// Partial - 所有属性可选
type PartialUser = Partial<User>;
function updateUser(user: User, updates: Partial<User>): User {
    return { ...user, ...updates };
}

// Required - 所有属性必需
type RequiredUser = Required<PartialUser>;

// Pick - 选取部分属性
type UserPreview = Pick<User, "id" | "name">;

// Omit - 排除部分属性
type UserWithoutId = Omit<User, "id">;

// Record - 创建对象类型
type PageInfo = Record<string, { title: string; url: string }>;

const pages: PageInfo = {
    home: { title: "首页", url: "/" },
    about: { title: "关于", url: "/about" }
};

const updated = updateUser({ id: 1, name: "A", email: "a@b.c", age: 20 }, { name: "B" });
console.log(updated.name);

    ❌ 常见错误

    Pick的键不正确：Pick<User, "phone"> 报错，"phone"不是User的键

    Partial类型安全：Partial后的属性都可能是undefined，使用前仍需判断

    📝 练习题

    1. 哪个工具类型可以将接口的所有属性变为可选？
    Partial<T>（如 type OptionalUser = Partial<User>）

    33. 条件类型

    📖 知识点讲解

    • 条件类型语法：T extends U ? X : Y，根据类型关系选择不同的结果类型
    • infer关键字：在条件类型中推断类型，如 T extends Array<infer U> ? U : T 提取数组元素类型
    • 分布式条件类型：当T是联合类型时，条件类型会自动分发到每个成员
    • 内置条件类型：Exclude、Extract、NonNullable、ReturnType、InstanceType等
    typescript
    // 基本条件类型
type IsString<T> = T extends string ? "yes" : "no";
type A = IsString<string>;  // "yes"
type B = IsString<number>;  // "no"

// infer关键字 - 提取类型
type ElementType<T> = T extends (infer E)[] ? E : T;
type Num = ElementType<number[]>;  // number
type Str = ElementType<string>;     // string

// ReturnType - 获取函数返回类型
function getUser() { return { name: "张三", age: 25 }; }
type UserType = ReturnType<typeof getUser>;
// { name: string; age: number }

// 内置条件类型
type T1 = Exclude<"a" | "b" | "c", "a">;      // "b" | "c"
type T2 = Extract<"a" | "b" | "c", "a" | "b">;  // "a" | "b"
type T3 = NonNullable<string | null | undefined>; // string

console.log("条件类型示例完成");

    ❌ 常见错误

    infer位置限制：infer只能在extends子句的条件类型中使用，不能在普通类型中使用

    分布式陷阱：type Wrap<T> = T extends any ? T[] : never 中联合类型会被分发，可能产生意外结果

    📝 练习题

    1. 如何提取函数返回值的类型？
    使用 ReturnType<typeof 函数名> 内置工具类型

    34. 映射类型

    📖 知识点讲解

    • 映射类型语法：{ [K in keyof T]: NewType }，基于旧类型创建新类型
    • 映射修饰符：使用+或-添加或移除readonly和可选修饰符
    • 键重映射：TS 4.1+支持 [K in keyof T as NewKey]: T[K]，可以变换键名
    • 模板字面量映射：结合模板字面量类型创建更灵活的映射
    typescript
    // 基本映射类型
type Readonlyify<T> = { readonly [K in keyof T]: T[K] };
type Optionalize<T> = { [K in keyof T]?: T[K] };

interface Info { name: string; age: number; }
type ReadonlyInfo = Readonlyify<Info>;
type OptionalInfo = Optionalize<Info>;

// 移除修饰符
type Mutable<T> = { -readonly [K in keyof T]: T[K] };
type Required2<T> = { [K in keyof T]-?: T[K] };

// 键重映射 (TS 4.1+)
type Getters<T> = {
    [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K]
};

interface Person { name: string; age: number; }
type PersonGetters = Getters<Person>;
// { getName: () => string; getAge: () => number }

// 条件映射
type NonNullableKeys<T> = {
    [K in keyof T]-?: T[K]
};

console.log("映射类型示例完成");

    ❌ 常见错误

    映射类型与接口混淆：映射类型只能使用type定义，不能使用interface

    键重映射语法：as 子句是TS 4.1+的特性，低版本不支持

    1. 写一个映射类型Stringify，将T的所有属性类型变为string？
    type Stringify<T> = { [K in keyof T]: string };

    35. 综合实战

    📖 知识点讲解

    • 项目架构：结合接口、泛型、装饰器、模块等构建完整TypeScript项目
    • 类型设计原则：优先使用unknown而非any、利用泛型提高复用性、使用字面量类型约束取值
    • 错误处理模式：使用可辨识联合表示操作结果（Result/Try模式），避免抛出异常
    • 类型安全API：利用条件类型、映射类型和模板字面量构建类型安全的API层
    typescript
    // 综合实战：类型安全的任务管理系统

// 1. 基础类型定义
type Result<T, E = string> =
    | { success: true; data: T }
    | { success: false; error: E };

interface Task {
    readonly id: string;
    title: string;
    description: string;
    status: "todo" | "in-progress" | "done";
    priority: "low" | "medium" | "high";
    createdAt: Date;
    tags: string[];
}

// 2. 泛型仓库
interface Repository<T extends { id: string }> {
    findAll(): T[];
    findById(id: string): T | undefined;
    create(item: Omit<T, "id">): T;
    update(id: string, data: Partial<T>): Result<T>;
    delete(id: string): Result<boolean>;
}

// 3. 实现仓库
class TaskRepository implements Repository<Task> {
    private tasks: Map<string, Task> = new Map();

    findAll(): Task[] { return [...this.tasks.values()]; }
    findById(id: string): Task | undefined { return this.tasks.get(id); }

    create(item: Omit<Task, "id">): Task {
        const task: Task = { ...item, id: crypto.randomUUID() };
        this.tasks.set(task.id, task);
        return task;
    }

    update(id: string, data: Partial<Task>): Result<Task> {
        const task = this.tasks.get(id);
        if (!task) return { success: false, error: "任务不存在" };
        const updated = { ...task, ...data, id: task.id };
        this.tasks.set(id, updated);
        return { success: true, data: updated };
    }

    delete(id: string): Result<boolean> {
        if (!this.tasks.has(id)) return { success: false, error: "任务不存在" };
        this.tasks.delete(id);
        return { success: true, data: true };
    }
}

// 4. 使用
const repo = new TaskRepository();
const task = repo.create({
    title: "学习TypeScript",
    description: "完成35章教程",
    status: "in-progress",
    priority: "high",
    createdAt: new Date(),
    tags: ["学习", "编程"]
});

console.log(`任务: ${task.title}, 状态: ${task.status}`);
const result = repo.update(task.id, { status: "done" });
if (result.success) {
    console.log(`完成: ${result.data.title}`);
}

    ❌ 常见错误

    过度设计类型：不要为了展示类型技巧而过度复杂化，类型设计应服务于业务需求

    忽略运行时验证：TypeScript类型只在编译时存在，API边界仍需运行时验证（如zod库）
