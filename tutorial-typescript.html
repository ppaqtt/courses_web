第1章 TypeScript 入门

▼

知识点

  • TypeScript 是 JavaScript 的超集，添加了静态类型系统
  • TypeScript 编译后生成纯 JavaScript，可在任何浏览器运行
  • TypeScript 提供类型检查、智能提示和更好的开发体验
  • 安装命令：npm install -g typescript
  • 编译命令：tsc filename.ts

代码示例

// hello.ts
function greet(name: string): string {
    return `Hello, ${name}!`;
}

const message: string = greet('TypeScript');
console.log(message);
// 编译命令
tsc hello.ts
// 生成 hello.js
node hello.js
// 输出: Hello, TypeScript!

常见错误

错误：未安装 TypeScript

运行 tsc 命令时报错 "command not found"，需要先安装 TypeScript：npm install -g typescript

错误：类型不匹配

传入参数类型与声明类型不一致会导致编译错误

练习题

  • 安装 TypeScript 并编译第一个 .ts 文件
  • 创建一个函数，接收两个数字参数并返回它们的和
  • 编写一个 TypeScript 程序，输出你的名字和年龄

第2章 基础类型

▼

知识点

  • string：字符串类型
  • number：数字类型（整数和浮点数）
  • boolean：布尔类型（true/false）
  • undefined：未定义值
  • null：空值
  • void：无返回值
  • any：任意类型（关闭类型检查）
  • never：永不返回的类型

代码示例

let name: string = 'Alice';

let isStudent: boolean = true;
let nothing: void = undefined;
let unknownValue: any = 'hello';
unknownValue = 42;

function warnUser(): void {
    console.log('This is a warning');
}

function throwError(): never {
    throw new Error('An error occurred');
}

class Car {

第10章 访问修饰符

▼

知识点

  • public：公共属性，默认值，可在任何地方访问
  • private：私有属性，只能在类内部访问
  • protected：受保护属性，可在类和子类中访问
  • 访问修饰符控制属性的可见性
  • TypeScript 编译后不保留访问修饰符

代码示例

class Person {
    public name: string;
    private age: number;
    protected email: string;
    
    constructor(name: string, age: number, email: string) {
        this.name = name;
        this.age = age;
        this.email = email;
    }
    
    public getAge(): number {
        return this.age;
    }
}

class Employee extends Person {
    constructor(name: string, age: number, email: string) {
        super(name, age, email);
    }
    
    public getEmail(): string {
        return this.email;
    }
}

const person = new Person('David', 35, 'david@example.com');
console.log(person.name);
console.log(person.getAge());

const employee = new Employee('Sarah', 28, 'sarah@company.com');
console.log(employee.getEmail());

private 属性不能在子类中访问，应使用 protected

练习题

第11章 抽象类

▼

知识点

  • 抽象类使用 abstract 关键字定义
  • 抽象类不能被实例化
  • 抽象方法必须在子类中实现
  • 抽象类可以包含具体方法和抽象方法
  • 抽象类用于定义基类的通用行为

代码示例

abstract class Shape {
    protected color: string;
    
    constructor(color: string) {
        this.color = color;
    }
    
    abstract calculateArea(): number;
    
    abstract calculatePerimeter(): number;
    
    getColor(): string {
        return this.color;
    }
}

class Circle extends Shape {
    private radius: number;
    
    constructor(color: string, radius: number) {
        super(color);
        this.radius = radius;
    }
    
    calculateArea(): number {
        return Math.PI * this.radius * this.radius;
    }
    
    calculatePerimeter(): number {
        return 2 * Math.PI * this.radius;
    }
}

class Rectangle extends Shape {
    private width: number;
    private height: number;
    
    constructor(color: string, width: number, height: number) {
        super(color);
        this.width = width;
        this.height = height;
    }
    
    calculateArea(): number {
        return this.width * this.height;
    }
    
    calculatePerimeter(): number {
        return 2 * (this.width + this.height);
    }
}

const circle = new Circle('red', 5);
console.log(circle.calculateArea());

常见错误

错误：尝试实例化抽象类

抽象类不能直接实例化，必须通过子类

错误：子类未实现抽象方法

子类必须实现父类的所有抽象方法

练习题

  • 创建一个抽象类 Vehicle
  • 定义抽象方法 start() 和 stop()

    swim(): void {

        console.log('Duck is swimming');

    getDepth(): number {

代码示例

function padLeft(value: string, padding: string | number): string {
    if (typeof padding === 'number') {
        return ' '.repeat(padding) + value;
    }
    return padding + value;
}

class Dog {
    bark(): void {
        console.log('Woof!');
    }
}

class Cat {
    meow(): void {
        console.log('Meow!');
    }
}

function makeSound(animal: Dog | Cat): void {
    if (animal instanceof Dog) {
        animal.bark();
    } else {
        animal.meow();
    }
}

    fly(): void;
    layEggs(): void;
}

interface Fish {
    swim(): void;
    layEggs(): void;
}

function move(animal: Bird | Fish): void {
    if ('fly' in animal) {
        animal.fly();
    } else {
        animal.swim();
    }
}

第16章 类型谓词

▼

知识点

  • 类型谓词是自定义的类型守卫函数
  • 使用 parameterName is Type 语法
  • 类型谓词函数返回 boolean
  • TypeScript 会根据返回值收窄类型
  • 类型谓词常用于复杂的类型检查

代码示例

interface Bird {
    fly(): void;
    layEggs(): void;
}

interface Fish {
    swim(): void;
    layEggs(): void;
}

function isBird(animal: Bird | Fish): animal is Bird {
    return (animal as Bird).fly !== undefined;
}

function move(animal: Bird | Fish): void {
    if (isBird(animal)) {
        animal.fly();
    } else {
        animal.swim();
    }
}

interface User {
    name: string;
}

interface Admin {
    name: string;
    isAdmin: boolean;
}

function isAdmin(user: User | Admin): user is Admin {
    return 'isAdmin' in user && (user as Admin).isAdmin;
}

function handleUser(user: User | Admin): void {

interface User {
    id: number;
    name: string;
    email?: string;
}

function getUser(id: number): User | null {
    const users: User[] = [
        { id: 1, name: 'Alice', email: 'alice@example.com' },
        { id: 2, name: 'Bob' }
    ];
    return users.find(u => u.id === id) || null;
}

function getUserName(user: User | null): string {
    return user?.name ?? 'Unknown';
}

type Result = { success: true; data: T } | { success: false; error: E };

function fetchData(url: string): Promise> {
    return fetch(url)
        .then(res => res.json())
        .then(data => ({ success: true, data }))
        .catch(error => ({ success: false, error }));
}
